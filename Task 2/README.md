# Task-2: Design & Integrate Your First Memory-Mapped IP

## Objective:- 
Design a simple memory-mapped IP, integrate it into the existing RISC-V SoC, and validate it through simulation.
Participants who already have the FPGA board may additionally validate the same IP on hardware, but hardware validation is optional in this task.

## Step 1: Understand the existing SoC
#### Identifying where memory-mapped peripherals are decoded
Inside the SOC module, 
```
wire isIO  = mem_addr[22];
wire isRAM = !isIO;
```
This Decodes IO.
When,
* mem_addr[22] = 0 -> access goes to RAM 
* mem_addr[22] = 1 -> access goes to IO peripherals

The memory address is converted into a word address using:
```
wire [29:0] mem_wordaddr = mem_addr[31:2];
```
The lower 2 address bits are removed because memory is word-aligned (32-bit words).

#### CPU Register Read/Write Mechanism
The CPU communicates with peripherals using the memory bus signals:
~~~
Signal                  Purpose
mem_addr	            Address generated by CPU
mem_wdata           	Data written by CPU
mem_wmask	            Write enable mask
mem_rdata	            Data returned to CPU
mem_rstrb	            Read request
~~~

#### Locating existing Memory-Mapped Peripherals
The existing address decode logic used:
```
localparam IO_LEDS_bit      = 0;
localparam IO_UART_DAT_bit  = 1;
localparam IO_UART_CNTL_bit = 2;
```
Each peripheral is selected using one-hot decoding on mem_wordaddr.
